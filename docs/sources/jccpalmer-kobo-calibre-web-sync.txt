## Calibre Web Admin Setup

1. Navigate to Admin settings > Configuration > "Edit Basic Configuration"
2. Expand Feature Configuration menu
3. Enable two options: "Enable Kobo sync" and "Proxy unknown requests to Kobo Store"
4. Verify Server External Port matches Calibre Web's port (default: 8083)
5. In user settings, check "Sync only books in selected shelves with Kobo"
6. Generate API token via "Create/View" button under Kobo Sync Token section

## Shelf Configuration

Create shelves and mark each with "Sync this shelf with Kobo device" checkbox to control what syncs to your reader.

## eReader.conf Modification

File path: .kobo/Kobo/Kobo eReader.conf
Section: [OneStoreServices]
Key: api_endpoint=

Replace the api_endpoint= line with the value obtained from Calibre-Web's Kobo Sync Token page.

## Key Gotchas

- External URLs may cause sync failures with reverse proxies or Cloudflare tunnels; use local IP address instead
- Store and Overdrive book covers often fail to display, showing generic placeholders instead
- Use HTTPS cautiously when tunneling through external services
- No Beta Features menu mentioned. No firmware version requirements stated.
- This only works on your local network unless Calibre-Web is exposed externally.
